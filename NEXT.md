## 推荐：优先做 #1 建筑场景功能

**核心理由：这是目前投入产出比最高的方向。**

1. **数据层已就绪，缺的是连接**：craftables.json5 已有15种装备配方含资源消耗定义；炼金材料（addAffix/replaceAffix/freezeAffix 等）已有完整概率表；但 `CraftState` 几乎是空壳，12种生产建筑在 location.dart 中全部 fall through 到 `default` 分支的 `onInteractLocationObject`，没有实际处理逻辑。

2. **它是其他系统的前置依赖**：
   - 装备打造 -> 灌灵（装备转卡牌）-> 丹药制作 -> 丹药卡牌，形成完整的 **制造-战斗循环链**
   - 经营策略（#2）需要建筑功能运转后才有意义
   - 门派关系（#3）的经济影响也需要贸易/建筑系统支撑

3. **最能直接提升可玩性**：玩家目前能进入据点看到建筑，但大部分点击后无事发生。补全这块后，游戏核心循环才成立。

## 关于卡牌、装备和丹药的不同打造体验

三条线的核心动词完全不同：

|              | 卡牌                        | 装备                            | 丹药                      |
| ------------ | --------------------------- | ------------------------------- | ------------------------- |
| **核心动词** | 微调                        | 拆装                            | 炼制                      |
| **获取**     | 卡包 / 问道碑（指定主词条） | 工坊制造（选类型+附加萃取材料） | 炼丹炉（选稀有度+主词条） |
| **改造**     | 6种材料精细操作             | 无（不满意就拆了重造）          | 无                        |
| **回收**     | 分解->碎片->合成打造材料    | 萃取->词条材料->下次制造用      | 无（消耗品用完即止）      |

装备的循环特别清晰：**造 -> 用 -> 拆 -> 造**。玩家不会在一件装备上反复折腾，而是不断用旧装备喂新装备，这和卡牌的"精修一张牌"体验完全不同。

一个细节确认：装备萃取时，是一件装备只能萃取出一条词条，还是每条额外词条都变成一个材料？建议**一件只出一条**（可配合锁定机制指定），否则高 rank 装备分解出 5-6 个词条材料，制造成本会迅速贬值。
