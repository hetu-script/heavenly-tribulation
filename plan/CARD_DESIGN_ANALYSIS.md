# 卡牌词条系统设计分析

## 当前系统概览

### 数据规模

- **主词条卡牌**: 65 张（通用 18 + 五大流派 47），另有 1 张绝世卡牌
- **额外词条**: 29 个（card_affixes.json5）
- **装备词条**: 来自 passives.json5，涵盖攻击、穿透、抗性、属性、资源上限、开场 buff 等
- **脚本函数**: 主词条 21 种 + 额外词条 12 种

### 战斗核心机制（已实现的亮点）

- 三层独立百分比乘算伤害公式
- 穿透系统（百分比无视防御）
- 阴阳资源体系（7 种正向 + 5 种负向能量）
- 正反制衡系统（强化/削弱互抵，防御/破甲互抵）
- 速度/闪避阈值触发（额外回合、跳过回合、无敌、蹒跚）
- 伤势系统（外/内/毒/幻觉四种）
- 护盾/结界硬防御层
- 基于优先级的回调执行链

## 〇、卡牌使用限制：从硬流派锁到软标签系统

### 现状

当前卡牌使用有双重限制：

| 层级       | 机制                                           | 实现位置                           |
| ---------- | ---------------------------------------------- | ---------------------------------- |
| 硬流派锁   | `checkRequirements()` 检查 `{genre}_rank` 被动 | `lib/logic/logic.dart`             |
| 资源软限制 | 卡牌消耗特定资源，资源生成不对称               | `enable_chakra/rage` 被动 + 天赋树 |

第二层已经构成了有效的软限制——没有 `enable_rage` 就无法被动生成怒气——硬流派锁是多余的。

### 推荐方案

**删除 `genre` 使用限制，改为标签（tag）系统。** 任何角色可使用任何标签的卡牌，但天赋树对该标签提供显著加成。

三层软限制：

| 层级       | 机制                 | 例子                                                     |
| ---------- | -------------------- | -------------------------------------------------------- |
| 资源不对称 | 不同流派生成不同资源 | 没有 `enable_rage` → 无法持续生成怒气 → 武术卡牌消耗不起 |
| 能量效率   | 天赋节点减费         | 本流派标签卡牌能量消耗 -1（最低 1）                      |
| 机制加成   | 标签专属被动         | 元素共鸣仅对 `元素` 标签卡牌生效                         |

### 五大标签体系

| 流派       | 标签   | 主要资源       | 资源生成方式                     |
| ---------- | ------ | -------------- | -------------------------------- |
| spellcraft | `元素` | 灵气（mana）   | talent + `mana` 卡               |
| swordcraft | `法器` | 剑气（chakra） | `enable_chakra`（造成伤害→剑气） |
| bodyforge  | `武术` | 怒气（rage）   | `enable_rage`（受到伤害→怒气）   |
| vitality   | `咒语` | 业力（karma）  | 战斗胜利积累                     |
| avatar     | `经文` | 混合           | 多途径                           |
| 通用       | 无标签 | 无特定         | 基础拳脚/防御，人人平等使用      |

### 优势

- **卡牌获取不挫败**：拿到非本流派卡牌可以 sub-optimally 使用，而非完全无用
- **构筑多样性**：跨标签混搭成为高级构筑选项
- **天赋树更有价值感**：从"解锁使用权"变为"让你比其他人用得好得多"
- **自然的新手引导**：天赋树引导新手走向流派专精，老手可以探索混搭

### 数据变更概要

- `cards.json5`：`genre` 字段保留用于卡包来源标注；新增 `tags` 字段承载标签
- `card.ht`：`BattleCard` 构造函数不再检查 `genre` 使用权限
- `logic.dart`：`checkRequirements()` 移除流派硬检查
- `passives.json5`：`{genre}_rank` 被动从纯标记扩展为携带标签加成效果
- 天赋节点：解锁对应流派标签的减费、资源生成、机制加成

## 一、攻击类型差异化

### 现状

目前徒手（拳脚）、武器、法术、诅咒四类攻击的本质区别为零——它们共享同一个 `attack` 函数，只是穿了不同的动画皮肤。`card_script.ht` 中的 `attack` 对所有 kind 做的是同一件事：播放动画 → `opponent.takeDamage()`。

问题不在于缺数值词条，而在于缺少**机制层面的差异**——不同类型的攻击应该在战斗流程中产生不同的连锁反应。

### 设计方案

#### 徒手（拳脚）→ "连击"节奏

徒手攻击速度快、不依赖装备。设计方向：多次轻击、连击累积、触发式收益。

- **拳（punch）**：基础伤害较低，但使用后获得 1 层急速（`speed_quick`）。配合已有的速度阈值系统（达到阈值获得额外回合），形成"越打越快"的正反馈
- **腿（kick）**：对血量低于 50% 的目标额外造成一次轻击，作为终结手段
- **擒拿（qinna）**：已有 `attack_slow_exhaust` 脚本，施加减速。可以进一步：若对手已有减速，额外施加外伤（`injury_external`）

#### 武器 → "架势与破防"

武器攻击伤害高但有装备依赖。设计方向：单次高伤、穿透防御、架招反击。

- **剑（sword）**：每次攻击获得少量剑气（`energy_positive_weapon`），类似"以战养气"
- **刀（sabre）**：单次伤害最高的武器类型，自带 20% 穿透——简单粗暴
- **枪（spear）**：攻击距离优势，伤害不受对手的闪避减伤（`dodge_invincible`）影响，或改为消耗其护盾
- **棍（staff）**：攻击后获得少量护盾（`shield_physical`），攻守兼备
- **弓（bow）**：远程特性，使用后本回合内对手的攻击伤害降低
- **暗器（dart）**：已有 `attack_multiple` 的多段攻击机制。可增加：每段命中施加 1 层外伤

#### 法术 → "元素共鸣"

法术消耗灵气，不同元素间有连锁互动。设计方向：元素组合、顺序规划、范围效果。

- **五行相生 subgame**：使用与上一张法术不同元素的法术时，额外获得 1 点灵气。这创造了一个资源规划小游戏——玩家需要安排法术使用顺序来最大化灵气效率
- **土（falling_stone）**：已有单次高伤害，可增加"对护盾造成双倍伤害"
- **火（fireball）**：已有单次高伤害，可增加"点燃"——燃烧对手 1 层生机（`energy_positive_life`）
- **风（wind_blade）**：已有减速。可额外：若本回合使用过火系法术，伤害提升（风助火势）
- **雷（lightning）**：已有 `attack_clumsy_exhaust`。可额外：若目标已有减速（`speed_slow`），额外触发一次伤害（导电）

#### 诅咒 → "延迟爆发与代价"

诅咒不直接追求单次高伤害，而是施加状态、延迟结算、操控对手资源。

- **咒术（sigil）**：已有 `attack_exhaust`。可增加：施加 1 层幻觉（`injury_hallucination`），对手回合开始时可能获得减速或笨拙
- **真言（power_word）**：已有降低/提升生命上限的效果。这个方向很对——诅咒牌的核心机制应该是"改变规则"而非"制造伤害"
- **符印/经文**：可设计为"延迟伤害"——伤害在对手回合结束时结算，但对手每有一种 debuff，伤害增加

#### 关键原则

不同类型之间的差异应该是**玩法的差异，不是数值的差异**。已有的速度阈值、穿透、正反制衡、debuff 系统尚未被充分利用来区分攻击类型——它们正是最好的素材。

## 二、装备与卡牌的互动

### 现状

装备→卡牌的关系是单向的：装备提供被动属性 → 卡牌在战斗中读取属性。缺少"装备改变卡牌行为"这一维度。

暗黑类游戏的精髓就在于此——传奇装备不只是加属性，而是改变技能的工作方式。这个游戏中卡牌就是"技能"，所以装备词条应该能够影响卡牌的结构、消耗和效果。

### 设计方案

装备对卡牌的影响可以分三个层次，由浅入深：

#### 层次一：数值调整（低成本实现）

| 词条效果                              | 说明                                                |
| ------------------------------------- | --------------------------------------------------- |
| "你的拳法卡牌基础伤害 +2"             | 针对特定 kind 的数值加成                            |
| "你的火系法术灵气消耗 -1（最低为 0）" | 针对特定元素的资源减免                              |
| "每回合开始时的抽牌数 +1"             | 影响手牌资源（当前通过 `getHandLimitForRank` 确定） |
| "手牌上限 +2"                         | 增加同时持有的卡牌数                                |

#### 层次二：行为修改（中等成本）

| 词条效果                                        | 说明                               |
| ----------------------------------------------- | ---------------------------------- |
| "你的拳法卡牌额外攻击一次，但伤害减半"          | 利好 on-hit 触发型构筑             |
| "你的火系法术改为造成物理伤害"                  | 伤害类型转换，让跨流派构筑成为可能 |
| "每当你获得护盾时，抽一张牌"                    | 防御转进攻的构筑方向               |
| "你的剑气上限翻倍，但每回合结束时失去 2 点剑气" | 高波动资源管理                     |
| "战斗开始时，随机将一张手牌的费用降为 0"        | 每场战斗的策略起点不同             |

#### 层次三：改变规则（高成本，适合绝世级装备）

| 词条效果                                                 | 说明                                             |
| -------------------------------------------------------- | ------------------------------------------------ |
| "你的每次攻击造成两种伤害类型（物理 + 精神）"            | 让对手无法用单一防御应对                         |
| "你的回合数不再限制手牌使用量"                           | 完全改变节奏，但需要其他限制（如翻倍的资源消耗） |
| "你的 buff 卡牌也可以造成伤害（伤害值 = buff 值的一半）" | 抹平攻击/增益的边界                              |

#### 实现路径

这些效果在技术上可以通过已有的 `ephemeralPassives` 系统实现——装备在战斗开始时注入一个临时被动（`isPermanent: false`），提供回调钩子，战斗结束后清除。

当前 `status_script.ht` 中的回调系统（`self_doing_damage`、`self_taken_damage` 等）已经为这类效果提供了挂载点。

## 三、绝世卡牌的设计方向

### 设计哲学

绝世卡牌类似《炉石传说》的橙卡——**不一定要数值最强，但一定要效果独特**。它们的核心价值在于：做到随机生成的普通卡牌**绝对做不到的事**。

普通卡牌的生成系统能做什么？

- 选择一个主词条（攻击/增益）
- 随机附加 0-6 个额外词条
- 额外词条从已有词条池中组合

因此，绝世卡牌应该探索这个系统**之外的领域**：

1. **跨界效果**：一张卡同时是攻击又是增益
2. **元游戏操作**：影响卡组中其他卡牌、影响抽牌堆、改变手牌
3. **规则改写**：临时改变某项战斗规则
4. **成长/演变**：效果随使用次数、回合数或境界变化
5. **高风险构筑核心**：强大的效果附带明显的代价或条件

### 设计示例

**参玄功**（已存在但脚本需修复）

> 效果：抽取 X 张卡牌。若其中有攻击牌，其本回合消耗的灵气减少 Y。
>
> 设计意图：抽牌 + 减费，类似炉石的"伺机待发"类效果。强大程度取决于你抽到了什么，每局体验不同。其 `uniqueId: "draw"` 意味着其他卡牌无法随机到抽牌效果——抽牌成为这张卡独有的能力。

**万剑归宗**（新设计，御剑流派）

> 效果：消耗你所有的剑气。每消耗 1 点剑气，对随机敌人造成 1 次伤害。
>
> 设计意图：资源倾泻型终结技。普通卡牌"消耗 N 点资源造成伤害"有固定 N，但"消耗全部资源，每点造成 X"是普通生成系统做不出的。配合剑气的生成体系（剑法攻击获得剑气 + 额外词条生成剑气），成为御剑卡组的核心终结手段。

**道心种魔**（新设计，炼魂流派）

> 效果：获得 5 层煞气。回合结束时，受到等于煞气层数的真实伤害。
>
> 设计意图：一张有代价的卡牌。本身看似负面，但可以配合"消耗煞气造成额外伤害"的额外词条，或配合"对手对你造成伤害时获得 debuff"的被动，形成高风险高回报的构筑。这创造了普通卡牌不敢做的设计空间——负收益卡牌。

**五行归一**（新设计，悟道流派）

> 效果：记录本局战斗中你使用过的元素种类数量。造成该数量 × 基础伤害的伤害。
>
> 设计意图：鼓励使用多种元素法术。游戏前期伤害低，后期随着使用过的元素种类增加而变强。配合"元素共鸣"的机制方向。

**不灭金身**（新设计，锻体流派）

> 效果：本回合获得无敌（`dodge_invincible`）。下回合失去全部怒气且怒气归零。
>
> 设计意图：一回合的无敌换取下回合的虚弱期。时机选择至关重要——在对手即将发动强力攻击时使用。

### 绝世卡牌 vs 普通卡牌的区别维度总结

| 维度     | 普通卡牌     | 绝世卡牌                                         |
| -------- | ------------ | ------------------------------------------------ |
| 词条来源 | 随机组合     | 固定设计                                         |
| 效果范围 | 数值加减     | 可改写规则                                       |
| 构筑角色 | 填充卡组     | 构筑核心                                         |
| 获取方式 | 卡包/掉落    | 特定条件                                         |
| 境界成长 | 通用升境规则 | 预设词条渐进解锁                                 |
| 构筑限制 | 可有多张     | 虽然可能多次获取，但卡组中只能有一张同名绝世卡牌 |

### 关于绝世卡牌境界提升的设计

延续上一轮讨论：绝世卡牌在数据中预设最高境界的完整词条列表——

```
uniquecard_example: {
  id: "uniquecard_example",
  isUnique: true,
  category: "attack",
  rank: 1,                    // 初始境界
  maxRank: 5,                 // 可达到的最高境界
  // 主词条（始终生效）
  valueData: [...],
  // 每个境界解锁的额外词条（按顺序）
  affixProgression: [
    { rank: 2, affix: "speed_quick" },
    { rank: 3, affix: "consume_chakra_for_extra_damage_physical" },
    { rank: 4, affix: "by_damage_heal" },
    { rank: 5, affix: "for_dexterity_increase_damage" },
  ],
}
```

升境时，新解锁的词条等级随机（可混元），但词条类型固定。这样既保留了绝世卡牌的"固定性"，又给玩家优化空间。

## 四、现存技术问题

1. **`uniquecard_draw` 的 `draw` 脚本是损坏的**（`card_script.ht` 中 `self.drawCards` 缺少 `()`）
2. **`rankIncrement` vs `increment` 不一致**：`uniquecard_draw` 使用 `rankIncrement`，但 `_updateAffixValue` 函数只读取 `increment`，意味着 `rankIncrement` 不会被计算
3. **`consume_vigor_heal` 额外词条脚本已实现但无数据引用**——死代码
4. **`maxLevel` 注释说用于限制速度/闪避类加成，但没有任何数据使用正值**，所有 `maxLevel` 都是 0

## 五、设计路线建议

建议分四个阶段推进，每个阶段产出可玩的内容：

### 第一阶段：攻击类型差异化（1-2 周）

**选一个流派作为试点**——建议先做**悟道 spellcraft**，因为它有最丰富的元素种类（土/火/风/雷/木 五种），最适合演示"元素共鸣"和"类型差异化"的设计理念。

具体工作：

- 修改悟道的 5 张攻击卡 + 5 张 buff 卡的脚本行为，赋予不同的机制特征
- 为悟道设计 3-5 个流派独有的额外词条（如元素共鸣、灵气溢出转化等）
- 这些变更直接在 `card_script.ht` 和 `card_affixes.json5` 中进行

### 第二阶段：第一批绝世卡牌（1 周）

- 为第一阶段实现的流派机制设计 2-3 张绝世卡牌
- 每张卡需要一个全新的、唯一的脚本行为（不能在 `CardScriptMain` 或 `CardScriptExtra` 中复用现有函数）
- 同时修复 `uniquecard_draw` 的脚本 bug，并统一 `rankIncrement`/`increment` 处理
- 实现绝世卡牌的境界提升系统（`affixProgression`）

### 第三阶段：装备 → 卡牌互动（1 周）

- 选 3-5 件装备，给它们添加"改变卡牌行为"的词条
- 扩展 `passives.json5` 的词条类型（需要新增 `isBattleCardModifier: true` 之类的标签）
- 确保战斗系统能在 `ephemeralPassives` 生命周期中正确处理这些效果
- 验证"战斗开始时注入 → 战斗中生效 → 战斗结束后清除"的完整链路

### 第四阶段：推广迭代（持续）

- 将悟道试点验证过的设计模式推广到锻体、御剑、炼魂三个流派
- 每个流派获得独有的机制特征 + 2-3 张绝世卡牌 + 流派特有额外词条
- 根据测试反馈调整平衡和趣味性

### 第五阶段：法身（最后实现）

法身区别于其他四个流派的核心特征是：**战斗中可以激活一个替代性的化身**。这不仅是外观变化（替换角色模型），而是用一套全新的角色数据临时替换当前角色：

- 激活化身后，角色的基础数值（生命上限、各项属性、资源上限）被化身数据覆盖
- 化身拥有独有天赋技能（类似装备主词条的特殊被动，但效果更强且具有流派辨识度）
- 化身可以改变可用卡牌的类型，或获得临时卡牌
- 化身的持续时间和冷却时间是关键平衡杠杆——例如"持续 3 回合，每场战斗只能激活一次"
- 取消化身时回归原始数据；化身期间消耗的资源和累积的状态是否保留，需要设计决策

法身与其他四流派的本质差异：

> 其他四流派：在已有角色框架内通过卡牌效果累积优势
> 法身：通过"变身"直接改写角色基础框架，是一种**爆发式节奏转变**——平时蓄力/积累资源，关键时刻降临化身，在有限的变
> 身回合内用碾压性的基础数值和特殊能力终结战斗。

技术挑战：

- 需要在战斗系统中支持运行时角色数据替换（`BattleCharacter` 预留 `transformCharacter(newData)` 方法）
- 化身的独有天赋技能需要一套独立的效果系统（可复用 `passives.json5` 的 `isEphemeral: true` 机制）
- 需要两个角色模型的美术资源（正常形态 + 化身形态），以及激活/取消时的动画过渡
- `characterCalculateStats()` 需支持数据源切换

法身放在最后实现是合理的——需要前四个阶段积累的脚本系统扩展、战斗运行时灵活性、以及美术资源作为基础。但在设计前四个流派时，可以为化身系统预留接口。

### 每个阶段的核心产出

| 阶段 | 可玩内容增量                            | 设计验证点                                     |
| ---- | --------------------------------------- | ---------------------------------------------- |
| 一   | 悟道流派的卡牌打出后"有感觉"了          | 元素共鸣机制是否创造了有趣的排序决策？         |
| 二   | 第一张真正意义上的绝世卡牌              | 绝世 vs 随机生成的区别是否足够显著？           |
| 三   | 装备不只是属性数字                      | 装备是否开始影响构筑方向（而非只是数值）？     |
| 四   | 四大流派各有特色（锻体/御剑/炼魂/悟道） | 玩家能否仅凭卡牌效果就判断出流派？             |
| 五   | 法身流派的"变身"机制                    | 化身带来的节奏转变是否创造了独特的构筑和打法？ |

## 六、额外词条体系待扩展（汇总）

### 条件触发类

- `if_opponent_has_debuff_extra_damage` — 对手有 debuff 时额外伤害
- `if_self_low_life_extra_defense` — 低生命时额外防御
- `if_first_card_this_turn_bonus` — 本回合第一张卡有加成
- `if_third_card_this_turn_bonus` — 本回合第三张卡有加成（鼓励连打）

### 资源转换类

- `convert_mana_to_chakra` — 将灵气转为剑气
- `spend_life_for_damage` — 消耗生命换伤害
- `overflow_mana_to_damage` — 灵气溢出量转为额外伤害
- `gain_rage_on_taking_damage` — 受到伤害获得怒气

### 行为变形类

- `change_damage_type_to_chi` — 将伤害类型改为真气
- `single_target_becomes_splash` — 单体伤害溅射
- `attack_gains_defense` — 攻击卡牌额外提供防御

### 流派专属基石类

- 每个流派 2-3 个，只能在对应流派的卡牌上出现
- 是构筑流派卡组的核心理由
- 数值应该显著高于通用词条，但范围受限（限定流派）
